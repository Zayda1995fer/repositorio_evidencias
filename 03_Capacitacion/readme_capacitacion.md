# 🎮 Actividad Práctica: Capacitación de un Nuevo Programador Junior en Git y GitHub

## 📖 Contexto

Has sido contratado como desarrollador dentro de una empresa de software que trabaja bajo metodologías ágiles y utiliza Git y GitHub para gestionar el código fuente de todos sus proyectos.

Recientemente se ha incorporado un nuevo integrante al equipo de desarrollo. Se trata de un **Programador Junior** que posee conocimientos básicos de programación, pero tiene poca experiencia trabajando con herramientas de control de versiones y colaboración de código.

Durante una reunión de planificación, el líder técnico del proyecto solicita al equipo diseñar una actividad práctica que permita al nuevo integrante aprender de manera rápida y divertida los conceptos fundamentales de Git y GitHub.

Uno de tus compañeros propone utilizar la plataforma:

```text
https://learngitbranching.js.org/?locale=es_ES
```

La cual presenta retos interactivos para aprender comandos de Git mediante simulaciones visuales.

Sin embargo, el líder del proyecto considera que esta herramienta es únicamente un punto de partida y solicita que el equipo diseñe una actividad complementaria que permita al nuevo programador aplicar los conocimientos adquiridos en un entorno más cercano a una situación real de desarrollo colaborativo.

---

# 🎯 Reto

Como integrante del equipo de desarrollo, deberás diseñar una propuesta de actividad práctica que ayude al nuevo Programador Junior a aprender Git y GitHub mediante la resolución de desafíos progresivos.

La actividad deberá considerar escenarios similares a los que se presentan en proyectos reales de desarrollo de software.

---

# 📋 Instrucciones

En equipo, desarrolla una propuesta que incluya los siguientes elementos:

## 1. Objetivo de aprendizaje

Definir claramente qué habilidades se espera que adquiera el Programador Junior.

Ejemplos:

* Clonar repositorios.
* Crear ramas.
* Realizar commits.
* Resolver conflictos simples.
* Crear Pull Requests.
* Revisar contribuciones de otros desarrolladores.

---

## 2. Descripción de la dinámica

Explicar cómo funcionará la actividad.

Algunas preguntas guía:

* ¿Será individual o colaborativa?
* ¿Se trabajará sobre un proyecto real o ficticio?
* ¿Existirán niveles o misiones?
* ¿Cómo se evaluará el progreso?

---

## 3. Retos o misiones

Diseñar una secuencia de actividades que incremente gradualmente la dificultad.

### Ejemplo

#### Nivel 1: Explorador Git

* Clonar un repositorio.
* Identificar la estructura del proyecto.
* Ejecutar la aplicación localmente.

#### Nivel 2: Constructor de Funcionalidades

* Crear una rama propia.
* Modificar un archivo.
* Registrar cambios mediante commits.

#### Nivel 3: Colaborador del Equipo

* Sincronizar cambios con el repositorio remoto.
* Resolver un conflicto sencillo.
* Crear un Pull Request.

#### Nivel 4: Revisor de Código

* Revisar el Pull Request de otro compañero.
* Aprobar o solicitar cambios.

---

## 4. Evidencias de aprendizaje

Definir qué evidencias deberán entregar los participantes.

Ejemplos:

* Capturas de pantalla.
* Historial de commits.
* Pull Requests realizados.
* Video demostrativo.
* Archivo README documentado.

---

## 5. Criterios de evaluación

Proponer una rúbrica simple para evaluar el desempeño del Programador Junior.

---

# 💡 Preguntas de Reflexión

Una vez diseñada la actividad, responder:

1. ¿Qué ventajas ofrece Git frente a guardar múltiples versiones de un archivo manualmente?
2. ¿Por qué es importante trabajar mediante ramas?
3. ¿Qué beneficios aportan los Pull Requests en un equipo de desarrollo?
4. ¿Cómo puede un líder técnico verificar que todos los integrantes están contribuyendo al proyecto?
5. ¿Qué habilidades técnicas y de colaboración desarrolla un programador al utilizar Git y GitHub?

---

# 📦 Entregable

El equipo deberá generar un documento en formato Markdown llamado:

```text
propuesta_capacitacion_git_github.md
```

que contenga:

* Objetivo de aprendizaje.
* Descripción de la dinámica.
* Retos propuestos.
* Evidencias.
* Criterios de evaluación.
* Respuestas a las preguntas de reflexión.

---

# 🏆 Desafío Extra

Investigar qué herramientas utilizan actualmente las empresas para complementar Git y GitHub, tales como:

* GitHub Projects.
* GitHub Issues.
* GitHub Actions.
* GitLab.
* Jira.
* Azure DevOps.

Y explicar brevemente cómo ayudan a mejorar la colaboración en equipos de desarrollo.

---

## 🚀 Meta Final

Diseñar una experiencia de aprendizaje que permita a un Programador Junior adquirir las competencias básicas necesarias para integrarse exitosamente a un equipo profesional de desarrollo de software que utiliza Git y GitHub como herramientas de trabajo colaborativo.
